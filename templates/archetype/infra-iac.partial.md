## 인프라/IaC 규칙
- 모든 인프라 변경은 코드로 관리 — 수동 콘솔 조작 금지
- plan/dry-run을 반드시 먼저 실행하고 확인 후 apply
- 상태 파일(tfstate 등)은 원격 백엔드에 저장, 로컬 커밋 금지
- 시크릿은 vault/secrets manager 사용 — 코드/상태 파일에 평문 금지
- 환경별(dev/staging/prod) 설정 분리 — 변수 파일 또는 워크스페이스
- 리소스 네이밍은 일관된 규칙 준수 (환경-서비스-리소스 등)
- drift 감지: 주기적으로 plan을 실행해 수동 변경 여부 확인
- 테스트: plan 성공 검증 + 모듈 단위 테스트 (terratest 등)
