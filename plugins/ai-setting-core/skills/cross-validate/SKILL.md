AI가 생성한 출력물을 실제와 대조합니다: $ARGUMENTS

## 검증 항목

### 스펙/문서 vs 코드
1. 문서에 적힌 클래스가 실제 코드에 존재하는지 (파일 검색)
2. 문서에 적힌 메서드가 실제 코드에 존재하는지
3. 문서에 적힌 파일 경로가 실제로 존재하는지
4. 문서의 의존성 목록 vs 실제 import

### 진행률/리포트 vs 실제 상태
1. 진행률 수치 vs 실제 완료된 항목 수
2. "변경된 파일" 목록 vs `git diff --name-only`
3. 비용/통계 수치 vs 실제 기록

### 요약 vs 원본
1. 세션 요약에 핵심 결정/실패가 누락되지 않았는지
2. 요약이 원본과 모순되는 내용을 담고 있지 않은지

## 불일치 분류

```
🔴 ERROR   — 심각한 불일치 → 다음 작업 진행 차단
🟡 WARNING — 경미한 불일치 → 자동 수정 시도 + 로그
🔵 INFO    — 사소한 차이 → 로그에만 기록
```

## 출력 형식

```
교차검증 결과:
🔴 ERROR (N건) — 진행 차단
  1. [파일]: [문서 내용] — [실제 상태]

🟡 WARNING (N건) — 자동 수정
  1. [파일]: [불일치 내용] (수정됨)

🔵 INFO (N건) — 기록만
  1. [파일]: [차이 내용]
```

## 원칙
- 의미 검증은 생성한 AI와 다른 호출로 수행 (자기 검증 금지)
- 구조적 검증(파일 존재, 클래스명)은 코드 검색으로 자동 대조
